对象实例追踪

Apptalking 可以生产环境中对JVM中任意的对象实例进行监控，统计其数量并发现其创建的代码位置。Apptalking对象实例追踪默认是关闭的状态，使用的时候需要开启。

对象实例列表

Apptalking 默认会监控 JVM 中实例数量最多的 Class，如果有其它比较重要的Class需要监控，可以在控制台上进行配置，Apptalking Agent 会对配置的 Class 进行监控。Apptalking 会统计 Class的实例数量和实例的总的体积。

对象创追踪

如果发现某个Class的实例数量异常，可以在Apptalking控制台执行 对象的创建追踪，Apptalking 会捕获创建对象的代码堆栈，并且关联事物，可以知道是哪个事物在什么代码位置造成的对象数量异常。
