错误

欢迎使用Apptalking，本文档将帮助您快速理解产品的核心功能,如果在使用过程当中遇到了问题，欢迎通过邮件的方式进行问答。

什么是错误

Apptalking中错误主要分为两大类：1、错误事物，2、异常。

  • 错误事物
    事物请求中如果有数据库、外部服务相关的异常，或者返回错误的Code，此次事物请求就会标示为错误的事物请求。
  • 异常
    应用中发生的异常信息，返回的错误Code，通过 Logger Framework 输出的Error级别的日志 都为异常内容。

错误事物

错误事物中有每分钟错误次数性能指标，快照列表为错误事物的快照信息，在快照中有具体的错误信息。

异常

  • 每分钟异常次数：每分钟异常发生的次数。
  • 错误Code：每分钟错误Code发生的次数。
  • 错误重定向：每分钟发生错误重定向的次数。

异常列表中有异常的名称和异常次数信息。

异常详情

  • 每分钟异常次数：此异常每分钟异常发生的次数。
    异常列表为此异常关联的错误事物快照。

异常堆栈

发生的异常信息中有详细的堆栈信息，可以便捷追溯异常发生的位置。

错误Code详情

错误Code按照URL进行统计发生的次数。

错误设置

  • 日志组件设置
    设置那些日志的组件进行监控。
  • 自定义日志组件
    如果Apptalking默认不支持的日志组件，用户自研的日志组件可使用自定义设置，使Apptalking识别监控。
  • 忽略异常
    如果不是重要的异常信息，可以通过配置忽略的异常内容，Apptalking 会放弃这些忽略的异常监控。
  • 忽略Categories / Loggers
    Apptalking不会监控配置了忽略的Categories / Loggers 中输入的日志信息。
  • HTTP Code
    配置自定义的错误Code范围。
  • 错误重定向
    配置错误重定向的页面。
